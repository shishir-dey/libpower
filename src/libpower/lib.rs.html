1//! # libpower - A no_std Rust library for power electronics, optimized for use on microcontrollers
2//!
3//! `libpower` is a comprehensive `no_std` Rust library designed for power electronics applications
4//! in embedded systems. It provides a collection of digital signal processing (DSP) algorithms,
5//! control systems, and power electronics primitives optimized for real-time control applications.
6//!
7//! ## Features
8//!
9//! - **No Standard Library Dependencies**: Built for embedded systems with `#![no_std]`
10//! - **Real-time Performance**: Optimized for deterministic, low-latency control loops
11//! - **Comprehensive Control Systems**: PID, PI, and advanced pole-zero controllers
12//! - **Digital Signal Processing**: High-performance filters (Butterworth, Chebyshev)
13//! - **Power Electronics Algorithms**: MPPT, PLL, coordinate transformations
14//! - **Signal Generation**: Configurable waveform generators with metrics
15//!
16//! ## Architecture
17//!
18//! The library is organized into several modules, each targeting specific aspects of
19//! power electronics control:
20//!
21//! ### Core Modules
22//!
23//! - [`battery`] - Battery management and State of Charge estimation
24//! - [`control`] - Digital control systems (PID, PI, pole-zero controllers)
25//! - [`filter`] - Digital signal processing filters
26//! - [`mppt`] - Maximum Power Point Tracking algorithms
27//! - [`pll`] - Phase-Locked Loop implementations
28//! - [`signal`] - Signal generation and analysis tools
29//! - [`transform`] - Coordinate transformations (Clarke, Park, etc.)
30//!
31//! ## Usage Patterns
32//!
33//! ### Basic Control Loop
34//!
35//! ```rust
36//! use libpower::control::cntl_pi::ControllerPI;
37//!
38//! // Create a PI controller with gains
39//! let mut controller = ControllerPI::with_gains(1.0, 0.1);
40//! controller.set_limits(-10.0, 10.0);
41//!
42//! // In your control loop
43//! let setpoint = 5.0;
44//! let measurement = 4.5;
45//! let output = controller.calculate(setpoint, measurement);
46//! ```
47//!
48//! ### Digital Filtering
49//!
50//! ```rust
51//! use libpower::filter::butterworth_lpf::ButterworthLPF;
52//!
53//! // Create and initialize a 4th-order Butterworth low-pass filter
54//! let mut filter = ButterworthLPF::<4>::new_uninit();
55//! filter.init(4, 1000.0, 44100.0); // 4th order, 1kHz cutoff, 44.1kHz sampling
56//!
57//! // Process samples
58//! let input_sample = 1.0;
59//! let filtered_output = filter.process(input_sample);
60//! ```
61//!
62//! ### MPPT Implementation
63//!
64//! ```rust
65//! use libpower::mppt::perturb_and_observe::MPPT;
66//!
67//! let mut mppt = MPPT::new();
68//! mppt.set_step_size(0.1);
69//! mppt.set_mppt_v_out_max(24.0);
70//! mppt.set_mppt_v_out_min(12.0);
71//!
72//! // In your control loop
73//! let pv_current = 5.0; // Example: 5A
74//! let pv_voltage = 18.0; // Example: 18V
75//! mppt.calculate(pv_current, pv_voltage);
76//! let reference_voltage = mppt.get_mppt_v_out();
77//! ```
78//!
79//! ### Battery State of Charge Estimation
80//!
81//! ```rust
82//! use libpower::battery::soc::Battery;
83//!
84//! let mut soc_estimator = Battery::new(0.8, 50.0 * 3600.0, 1.0);
85//! soc_estimator.set_process_noise(1e-5);
86//! soc_estimator.set_measurement_noise(5e-4);
87//!
88//! // In your control loop
89//! let battery_current = -15.0; // 15A discharge
90//! let terminal_voltage = 3.6; // Measured terminal voltage
91//! let estimated_soc = soc_estimator.update(battery_current, terminal_voltage);
92//! ```
93//!
94//! ## Design Principles
95//!
96//! ### Performance
97//! - All algorithms use single-precision floating-point arithmetic (`f32`)
98//! - Minimal memory allocation (stack-based, pre-allocated buffers)
99//! - Optimized for real-time control loops (typically 1-100 kHz)
100//!
101//! ### Safety
102//! - Saturation limits and bounds checking where appropriate
103//! - Graceful handling of edge cases and invalid inputs
104//! - Clear error reporting through `Result` types where applicable
105//!
106//! ### Modularity
107//! - Each algorithm is self-contained with minimal dependencies
108//! - State variables are encapsulated within structs
109//! - Configurable parameters with sensible defaults
110//!
111//! ## Target Applications
112//!
113//! - DC-DC converters and power supplies
114//! - Motor drives and inverters
115//! - Solar inverters and MPPT controllers
116//! - Grid-tied power electronics
117//! - Battery management systems
118//! - Power factor correction circuits
119//!
120//! ## Examples
121//!
122//! See the individual module documentation and the `tests/` directory for comprehensive
123//! examples of each algorithm and typical usage patterns.
